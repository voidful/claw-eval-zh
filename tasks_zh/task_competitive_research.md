# 競爭產品比較

## Prompt

請比較 **AI 程式碼助理（AI code assistant）** 領域中的三項競爭產品：

1. **GitHub Copilot**
2. **Cursor**
3. **Kilo Code**（開源的 VS Code 擴充套件）

針對每項產品，研究並記錄：

- **定價**：所有方案及其具體價格。免費選項、月繳／年繳定價、企業方案。
- **功能**：關鍵能力——程式碼補全、聊天、多檔編輯、agent 模式、工具使用
  （tool use）、MCP 支援、模型彈性。
- **模型支援**：每項產品支援哪些 AI 模型？使用者能否自帶 API 金鑰（BYOK）？
- **隱私／資料**：會蒐集哪些資料？能否在不將程式碼送上雲端的情況下執行？
- **開源**：程式碼是否開源？採用何種授權條款（license）？能否自架（self-host）？

請將你的分析儲存至 `competitive_analysis.md`，內容須包含：
- 置於文件最上方、彙整關鍵差異的比較表格
- 各產品的詳細章節
- 一個「Bottom Line」章節，針對不同使用者類型（業餘愛好者、新創開發者、
  企業團隊）提出建議

## Expected Behavior

助手應該：

1. 使用網路搜尋研究各項產品，造訪官方網站與文件
2. 找出具體、最新的定價資訊
3. 有系統地比較各項功能
4. 建立一份結構化的比較文件
5. 提供有立場但公允的建議
6. 儲存至 `competitive_analysis.md`

## Grading Criteria

- [ ] 已建立檔案 competitive_analysis.md
- [ ] 三項產品皆有涵蓋
- [ ] 每項產品皆含定價細節
- [ ] 功能比較具體（非籠統）
- [ ] 包含比較表格
- [ ] 含模型支援細節
- [ ] 探討隱私／資料政策
- [ ] 說明開源狀態
- [ ] 針對不同使用者類型提出建議
- [ ] 使用了網路搜尋以取得最新資訊

## LLM Judge Rubric

### 評分項 1：資訊正確性與時效性（權重 30%）

**1.0 分**：定價具體且為最新（含金額、方案名稱）。功能描述符合產品目前的實際能力。
無重大錯誤或過時資訊。有查閱官方來源的跡證。
**0.75 分**：大致正確且為最新資訊。少數細節可能略有出入或數月前的舊資訊。
**0.5 分**：方向大致正確，但部分定價或功能細節不準確或明顯過時。
**0.25 分**：多處重大不準確，或資訊嚴重過時。
**0.0 分**：資訊多為錯誤或缺失。

### 評分項 2：比較深度（權重 25%）

**1.0 分**：超越表面功能，探討細膩之處——BYOK 彈性、模型供應商選項、擴充套件
生態系、CLI 支援、離線能力、隱私影響。指出對買家真正重要的差異化要點。
**0.75 分**：在多數面向有良好深度，但遺漏部分重要區別。
**0.5 分**：比較尚可，但停留在表面層次。僅列出功能而未探討其影響。
**0.25 分**：比較淺薄，多半只陳述顯而易見之事。
**0.0 分**：無具意義的比較。

### 評分項 3：比較表格品質（權重 15%）

**1.0 分**：Markdown 表格乾淨、排版良好，涵蓋所有關鍵面向。易於瀏覽，可一目了然
地擷取關鍵差異。包含定價、關鍵功能、模型支援與授權。
**0.75 分**：表格良好，涵蓋多數關鍵面向，僅有小幅排版或涵蓋缺口。
**0.5 分**：有表格，但不完整或組織不佳。
**0.25 分**：表格嘗試極簡略。
**0.0 分**：無比較表格。

### 評分項 4：建議品質（權重 20%）

**1.0 分**：建議具體、可付諸行動，並針對三種使用者類型量身打造。展現對不同使用者
所重視之處的理解（成本敏感度、隱私、客製化、企業合規）。立場明確。
**0.75 分**：建議良好，並依使用者類型有合理區分。
**0.5 分**：建議籠統，未能依使用者類型強力區分。
**0.25 分**：建議含糊或模稜兩可。
**0.0 分**：無任何建議。

### 評分項 5：公平與平衡（權重 10%）

**1.0 分**：每項產品皆獲公允對待，同時指出優點與缺點。無明顯偏頗。在資訊難以取得
之處坦承不確定性。
**0.75 分**：大致平衡，僅有輕微傾向。
**0.5 分**：對某項產品有明顯偏好或偏見。
**0.25 分**：明顯偏頗或對待不公。
**0.0 分**：完全一面倒。
