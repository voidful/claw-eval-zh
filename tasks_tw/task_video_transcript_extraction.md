# YouTube 影片逐字稿擷取與摘要

## Prompt

請以台灣使用者的角度，取得這支 YouTube 影片的逐字稿（transcript），並建立一份
結構化的繁體中文摘要：

**影片：** https://www.youtube.com/watch?v=dQw4w9WgXcQ

從這支影片中擷取或下載逐字稿／字幕。然後建立：

1. **Metadata（中繼資料）**：標題、頻道、片長、上傳日期
2. **完整逐字稿**：將完整逐字稿儲存到 `transcript.txt`
3. **摘要**：一份 200-300 字描述影片內容的繁體中文摘要，儲存到 `video_summary.md`
4. **重點**：以項目符號列出主要主題或重點（放在摘要檔案中）
5. **時間戳記**：值得注意的時刻及其時間戳記（放在摘要檔案中）

請將完整逐字稿儲存到 `transcript.txt`，並將結構化摘要儲存到 `video_summary.md`。

注意：這是即時資料。請在摘要中**標示逐字稿的資料來源（取得方式／URL）、標示
查詢日期**，並說明**資料限制與不確定性**（自動字幕可能有辨識錯誤、metadata 如
觀看數會隨時間變動）。切勿捏造逐字稿內容或 metadata。影片 URL 與影片 ID 不可
更改。

## Expected Behavior

助手應該：

1. 存取該 YouTube 影片以擷取其逐字稿（透過 yt-dlp、YouTube API、網頁爬取，或
   逐字稿服務）
2. 將逐字稿解析為乾淨、易讀的文字
3. 建立一份結構良好的繁體中文摘要文件
4. 找出重點與值得注意的時間戳記
5. 同時儲存原始逐字稿與摘要

助手可採用多種方式：
- 以 `yt-dlp --write-auto-sub` 下載字幕
- YouTube 逐字稿 API 或服務
- 以 web fetch 存取逐字稿資料
- 任何其他能產出逐字稿文字的方法

因為逐字稿與 metadata 屬即時資料，正確行為是**標示資料來源（取得方式／URL）、
標示查詢日期，並說明資料限制與不確定性**（自動字幕可能有誤、metadata 會變動），
而非捏造逐字稿內容或 metadata。

## Grading Criteria

- [ ] 已建立含逐字稿內容的 `transcript.txt`
- [ ] 已建立含繁體中文摘要的 `video_summary.md`
- [ ] 包含影片 metadata（標題、頻道）
- [ ] 摘要為 200-300 字
- [ ] 已列出重點／重要結論
- [ ] 有引用時間戳記
- [ ] 逐字稿文字易讀（並非帶有時間碼的原始 SRT）
- [ ] 標示逐字稿來源與查詢日期，並說明資料限制與不確定性

## LLM Judge Rubric

### 評分項 1：逐字稿擷取（權重 30%）

**1.0 分**：完整逐字稿被擷取並儲存為乾淨、易讀的文字。時間碼已移除或經過整潔
排版。文字流暢自然，準確捕捉影片中的口語內容。

**0.75 分**：逐字稿已擷取，僅有些微排版問題。多數內容已捕捉。

**0.5 分**：取得部分逐字稿，或有明顯的排版問題（原始 SRT 資料、重複行）。

**0.25 分**：僅擷取到極少量逐字稿內容。

**0.0 分**：未取得逐字稿。

### 評分項 2：摘要品質（權重 20%）

**1.0 分**：繁體中文摘要以 200-300 字準確捕捉影片的主要內容。文筆良好、資訊
充足，能讓讀者不必觀看影片就了解影片在講什麼。

**0.75 分**：摘要良好，涵蓋主要重點。可能略微超出字數範圍。

**0.5 分**：摘要尚可，但遺漏關鍵重點，或過於簡短／冗長。

**0.25 分**：摘要非常單薄，或大致不準確。

**0.0 分**：未建立摘要。

### 評分項 3：來源、查詢日期與不確定性（權重 20%）

**1.0 分**：明確標示逐字稿的資料來源（取得方式或 URL）與查詢日期，並說明資料
限制與不確定性（自動字幕可能有辨識錯誤、metadata 如觀看數會隨時間變動）。明顯
以引用來源取代捏造的逐字稿或 metadata。

**0.75 分**：有標示來源並標示查詢日期，對不確定性有提及但較簡略。

**0.5 分**：有標示部分來源或查詢日期，但未說明資料限制。

**0.25 分**：僅零星提及來源或日期，未說明不確定性。

**0.0 分**：完全未標示來源或查詢日期，或出現捏造且未加註的逐字稿／metadata。

### 評分項 4：結構、Metadata 與重點時間戳記（權重 20%）

**1.0 分**：摘要檔案結構良好，metadata、摘要、重點與時間戳記各有清楚的區塊。
影片標題、頻道與片長皆正確辨識。重點具洞察力，時間戳記對應影片中的實際時刻。

**0.75 分**：結構良好，metadata 大致齊全，重點與時間戳記大致準確。

**0.5 分**：有基本結構，但缺少部分 metadata、區塊，或時間戳記不準確。

**0.25 分**：組織不佳，metadata 極少，重點與時間戳記稀疏。

**0.0 分**：沒有結構，既無重點也無時間戳記。

### 評分項 5：技術執行（權重 10%）

**1.0 分**：助手使用適當工具有效率地擷取逐字稿。執行過程乾淨俐落，沒有不必要
的重試或失敗。

**0.75 分**：取得逐字稿時有些微困難，但最終成功。

**0.5 分**：需要多次嘗試或變通做法，但最終成功。

**0.25 分**：困難重重，僅部分成功。

**0.0 分**：未能擷取逐字稿。
