# 台灣 SaaS／雲端服務定價比較

## Prompt

你的團隊需要為一個正式環境（production）應用程式選擇一項 **SaaS／雲端服務**。
請比較以下幾家在台灣可採購之雲端／SaaS 供應商的定價，金額一律以**新臺幣
（NT$）**呈現（外幣請註明匯率換算基礎與日期）：

1. **Amazon Web Services（AWS）**
2. **Microsoft Azure**
3. **Google Cloud Platform（GCP）**
4. **中華電信 hicloud**
5. **Linode／Akamai 或其他在台可用之雲端服務**

為了進行標準化比較，請為兩種配置估價：

**Config A（Small）：** 2 vCPUs、8 GB RAM、100 GB 儲存空間、單一可用區
**Config B（Production）：** 4 vCPUs、16 GB RAM、500 GB 儲存空間、高可用性

針對每家供應商，記錄 Config A 與 Config B 的每月預估費用（新臺幣）、包含哪些
項目、定價模式、隱藏成本（資料傳輸、IOPS 等），以及是否提供免費方案或試用。

請將你的分析儲存至 `pricing_comparison.md`，內容須包含置於最上方的定價比較
表格、各供應商的詳細拆解、點出隱藏成本的總體擁有成本（TCO）分析，以及一份
附理由的建議。

注意：這是即時資料。請對每月費用、機型／SKU 與匯率**標示資料來源、標示查詢
日期**，並說明**資料限制與不確定性**（雲端定價常變動、匯率波動、區域差異）。
切勿捏造或寫死特定價格數字。

## Expected Behavior

助手應該：

1. 從各供應商的定價頁面研究目前的定價，並換算為新臺幣（註明匯率與日期）
2. 將兩種配置對應到最接近的可用機型（instance types）
3. 計算含相關附加項目的每月費用
4. 找出隱藏或容易被忽略的成本
5. 建立一份結構化的比較文件
6. 儲存至 `pricing_comparison.md`

因為雲端定價屬即時資料，正確行為是**標示資料來源、標示查詢日期，並說明資料
限制與不確定性**（價格與匯率隨時間、區域變動），而非捏造或寫死特定價格。

## Grading Criteria

- [ ] 已建立檔案 pricing_comparison.md
- [ ] 5 家供應商皆有涵蓋（含至少一家在台供應商）
- [ ] 為每家估算 Config A 的價格（新臺幣）
- [ ] 為每家估算 Config B 的價格（新臺幣）
- [ ] 找出隱藏成本
- [ ] 包含定價表格
- [ ] 提及機型／SKU
- [ ] 包含免費方案資訊與附理由的建議
- [ ] 標示資料來源、查詢日期與匯率換算基礎，並說明資料限制與不確定性

## LLM Judge Rubric

### 評分項 1：定價正確性（權重 30%）

**1.0 分**：每月費用估算具體（含金額、以新臺幣呈現），且落在目前市場行情的
合理範圍內。有指明機型或 SKU。估算涵蓋運算、儲存與 I/O。明顯取自定價頁面。

**0.75 分**：估算合理並指明機型。有小幅不準確，或遺漏部分成本項目。

**0.5 分**：估算的數量級正確，但缺乏具體性。定價籠統，未指明機型。

**0.25 分**：估算含糊，或定價明顯過時。

**0.0 分**：無定價資訊，或估算極度不準確。

### 評分項 2：隱藏成本分析（權重 20%）

**1.0 分**：找出各供應商特有的隱藏成本——資料傳輸出口（egress）、IOPS 費用、
超出免費額度的備份保留、監控附加項目、跨區複寫成本。並至少量化其中部分項目。

**0.75 分**：找出數項隱藏成本，並有部分量化。

**0.5 分**：提及存在隱藏成本，但未量化或流於含糊。

**0.25 分**：對額外成本幾乎未著墨。

**0.0 分**：未探討隱藏成本。

### 評分項 3：來源、查詢日期與不確定性（權重 20%）

**1.0 分**：所有價格、機型／SKU 與匯率換算皆標示可追溯的資料來源與查詢日期，
並說明資料限制與不確定性（雲端定價常變動、匯率波動、區域差異）。明顯以引用
來源取代捏造或寫死的價格。

**0.75 分**：多數價格有標示來源並標示查詢日期，對匯率與不確定性有提及但較簡略。

**0.5 分**：有標示部分來源或查詢日期，但不一致，或未說明資料限制與匯率基礎。

**0.25 分**：僅零星提及來源或日期，未說明不確定性。

**0.0 分**：完全未標示來源或查詢日期，或出現捏造／寫死且未加註的價格。

### 評分項 4：比較完整度與呈現（權重 20%）

**1.0 分**：兩種配置皆為 5 家供應商完成估價（以新臺幣計）。包含哪些項目內含
vs. 哪些需額外付費。乾淨的比較表格讓人能一目了然地比較，可實際用於採購決策。

**0.75 分**：多數供應商皆完成兩種配置的估價，表格與組織良好，僅有小幅缺口。

**0.5 分**：一種配置涵蓋良好，另一種單薄；或資訊難以跨供應商比較。

**0.25 分**：涵蓋有重大缺口，組織不佳。

**0.0 分**：涵蓋少於 3 家供應商，或無比較結構。

### 評分項 5：建議品質（權重 10%）

**1.0 分**：建議考量總體擁有成本，而不只是表面價格。依使用情境區分（新創 vs.
企業），並坦承權衡取捨（價格 vs. 功能 vs. 維運簡易度、在台支援）。

**0.75 分**：建議良好且附理由。細膩度有小幅缺口。

**0.5 分**：建議籠統，缺乏強有力的理由。

**0.25 分**：建議含糊或缺乏佐證。

**0.0 分**：無任何建議。
