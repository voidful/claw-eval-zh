# 台灣 AI 治理與資料保護規範研究

## Prompt

有一家打造 AI 驅動開發者工具的 SaaS 公司，需要了解其在**台灣 AI 治理與資料
保護相關規範**下的義務。他們在台灣提供服務、蒐集使用者個資，並透過 API 部署
模型。

請研究台灣相關法規與政策，並製作一份合規簡報。你的報告應涵蓋：

1. **個人資料保護法（個資法）**：蒐集、處理與利用個人資料的義務為何？告知
   義務、當事人權利、特種個資、跨境傳輸限制有哪些？
2. **資通安全管理法（資安法）**：適用對象為何？該公司若為非公務機關，在何種
   情況下會被納管？有哪些資安維護義務？
3. **AI 治理框架／指引**：台灣目前推動中的 AI 治理框架、指引或草案（例如人工
   智慧基本法草案、行政機關 AI 應用指引）現況為何？
4. **主管機關**：哪些政府機關負責個資保護與 AI 治理？
5. **跨境資料傳輸**：將個資或使用者資料傳送至境外（例如透過第三方 LLM API）
   有哪些限制與應注意事項？
6. **罰則**：違反個資法或資安法的罰則為何？
7. **實務步驟**：該公司現在應採取哪些行動以準備合規？

請將報告儲存至 `ai_governance_briefing.md`。請盡可能
引用法規的具體條文或正式名稱。

注意：這是即時資料。請對法規條文、罰則與政策現況**標示資料來源、標示查詢
日期**，並說明**資料限制與不確定性**（法規可能修訂、AI 治理框架仍在發展中）。
切勿捏造條文編號或罰則金額。法規正式名稱（個人資料保護法、資通安全管理法）
保留原文。

## Expected Behavior

助手應該：

1. 使用網路搜尋及／或官方來源研究台灣個資法、資安法與 AI 治理相關規範
2. 找出具體的法規條文與關鍵條款
3. 正確說明蒐集／處理個資的義務、跨境傳輸限制與資安義務
4. 找出與使用第三方 LLM API 相關的跨境傳輸考量
5. 製作一份可付諸行動的合規簡報
6. 儲存至 `ai_governance_briefing.md`

因為法規與 AI 治理政策可能修訂、仍在發展中，正確行為是**標示資料來源、標示
查詢日期，並說明資料限制與不確定性**，而非捏造條文編號或罰則金額。

## LLM Judge Rubric

### 評分項 1：法律正確性（權重 30%）

**1.0 分**：個人資料保護法與資通安全管理法的主要義務描述正確。條文或關鍵概念
引用正確（如告知義務、當事人權利、跨境傳輸限制、資安維護計畫）。罰則金額與
法規一致。AI 治理框架現況描述正確。

**0.75 分**：大致正確，整體框架無誤。條文編號或具體門檻有小幅不準確。

**0.5 分**：整體框架正確，但多項具體細節錯誤或缺漏。

**0.25 分**：有重大不準確，或將台灣法規與其他司法管轄區法規混淆。

**0.0 分**：缺少報告，或根本上不準確。

### 評分項 2：實務相關性（權重 20%）

**1.0 分**：報告明顯針對該公司的處境（AI 開發者工具、API 部署、在台蒐集個資）
量身打造。跨境傳輸分析具體考量透過第三方 LLM API 將資料送出境的情境。實務
步驟具體且可付諸行動。

**0.75 分**：與該公司處境的相關性良好。實務步驟有用，但可更具體。

**0.5 分**：涵蓋了法規，但未強力連結到該公司的具體處境。

**0.25 分**：籠統的法規概述，未做任何量身打造。

**0.0 分**：與所述情境不相關。

### 評分項 3：來源、查詢日期與不確定性（權重 20%）

**1.0 分**：所有條文引用、罰則金額與政策現況皆標示可追溯的資料來源（官方法規
資料庫、主管機關公告等）與查詢日期，並說明資料限制與不確定性（法規可能修訂、
AI 治理框架仍在發展中）。明顯以引用來源取代捏造或寫死的內容。

**0.75 分**：多數關鍵內容有標示來源並標示查詢日期，對不確定性有提及但較簡略。

**0.5 分**：有標示部分來源或查詢日期，但不一致，或未說明資料限制。

**0.25 分**：僅零星提及來源或日期，未說明不確定性。

**0.0 分**：完全未標示來源或查詢日期，或出現捏造／寫死且未加註的條文或罰則。

### 評分項 4：跨境傳輸與資安涵蓋度（權重 20%）

**1.0 分**：徹底涵蓋跨境個資傳輸的限制與注意事項（主管機關得限制、告知與同意、
委外處理監督），以及資通安全管理法下的資安維護義務，並說明以 API 為基礎使用
第三方模型如何影響合規。

**0.75 分**：跨境傳輸與資安涵蓋良好，處理了主要義務。

**0.5 分**：提及跨境傳輸或資安，但欠缺深度。

**0.25 分**：跨境傳輸或資安著墨極少。

**0.0 分**：未處理跨境傳輸與資安義務。

### 評分項 5：時程、結構與引用品質（權重 10%）

**1.0 分**：報告組織良好、章節清楚、易於瀏覽，可作為參考文件使用。引用法規的
具體條文或正式名稱（如「個人資料保護法第 X 條」），引用增添可信度與可追溯性。

**0.75 分**：結構與引用良好，有小幅缺口。

**0.5 分**：結構尚可，引用有限。

**0.25 分**：結構不佳，具體參照極少。

**0.0 分**：無可辨識的結構，亦無引用或條文參照。
